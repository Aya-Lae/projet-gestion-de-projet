# 📋 Mini Trello Clone — Gestion de Projets

A full-stack Kanban-style project management web app built with **Spring Boot 3** (backend) and **React + TypeScript + Vite** (frontend). Team project.

---

## 🛠️ Tech Stack

| Layer | Technology |
|---|---|
| Frontend | React 18, TypeScript, Vite, Tailwind CSS, shadcn/ui |
| Drag & Drop | @hello-pangea/dnd |
| HTTP Client | Axios + TanStack Query |
| Backend | Spring Boot 3.2, Java 17 |
| Security | Spring Security + JWT (jjwt 0.11.5) |
| Database | MySQL (via Spring Data JPA / Hibernate) |
| Build Tool | Maven |

---

## ✨ Features

- 🔐 **Authentication** — Register, login, and logout with JWT-based auth
- 📁 **Projects** — Create, update, and delete projects with a name, description, and color label
- 👥 **Team members** — Add/remove members on each project
- ✅ **Tasks (Kanban)** — Create and manage tasks per project across three columns: `TODO`, `IN_PROGRESS`, `DONE`
- 🖱️ **Drag & drop** — Move tasks between columns with @hello-pangea/dnd
- 📄 **Task details** — Edit title, description, assignee, and due date; add comments
- 🌙 **Theme** — Light/dark mode support via next-themes
- 📊 **Dashboard** — Overview of your projects and tasks

## ⚙️ Getting Started

### Prerequisites

- Java 17+
- Maven
- Node.js 18+ and npm
- MySQL (XAMPP or standalone)

---

### 🗄️ Database Setup

1. Start MySQL and create a database named `projectdb`.
2. The schema is auto-generated by Hibernate on first run (`ddl-auto=update`).

---

### 🔧 Backend Setup

```bash
cd backend

# Copy and configure the properties file
# Edit src/main/resources/application.properties:
#   spring.datasource.url=jdbc:mysql://localhost:3306/projectdb
#   spring.datasource.username=root
#   spring.datasource.password=your_password

# Run the Spring Boot app
./mvnw spring-boot:run
```

The API will be available at `http://localhost:8080`.

---

### 💻 Frontend Setup

```bash
cd "front end"

npm install

npm run dev
```

The app will be available at `http://localhost:5173`.

> The frontend talks to the backend at `http://localhost:8080/api` by default. Override it by setting `VITE_API_URL` in a `.env` file.

## 🔌 API Endpoints

### Auth — `/api/auth`
| Method | Endpoint | Description |
|---|---|---|
| POST | `/api/auth/register` | Register a new user |
| POST | `/api/auth/login` | Login and receive a JWT |
| POST | `/api/auth/logout` | Logout (invalidate token) |
| GET | `/api/auth/me` | Get current authenticated user |

### Projects — `/api/projects`
| Method | Endpoint | Description |
|---|---|---|
| GET | `/api/projects` | Get all projects for the current user |
| GET | `/api/projects/{id}` | Get a project by ID |
| POST | `/api/projects` | Create a new project |
| PUT | `/api/projects/{id}` | Update a project |
| DELETE | `/api/projects/{id}` | Delete a project |
| POST | `/api/projects/{id}/members` | Add a member to a project |
| DELETE | `/api/projects/{id}/members/{name}` | Remove a member from a project |

### Tasks — `/api/projects/{projectId}/tasks`
| Method | Endpoint | Description |
|---|---|---|
| GET | `/api/projects/{projectId}/tasks` | Get all tasks for a project |
| POST | `/api/projects/{projectId}/tasks` | Create a task |
| PUT | `/api/projects/{projectId}/tasks/{taskId}` | Update a task |
| DELETE | `/api/projects/{projectId}/tasks/{taskId}` | Delete a task |
| POST | `/api/projects/{projectId}/tasks/{taskId}/comments` | Add a comment to a task |

> All endpoints except `/api/auth/register` and `/api/auth/login` require a `Authorization: Bearer <token>` header.
